How it works

  1. 01

    Sign in with Google

    Google issues an ID token. Core Java verifies the signature, audience, and expiry — then mints a signed session cookie.

  2. 02

    Paste or scan

    Type a number, or scan an image. OCR runs locally in your browser via Tesseract.js; the image is never uploaded.

  3. 03

    Get a deterministic verdict

    Only the text candidate is sent to the Core Java validator for structure and the Aadhaar Verhoeff checksum.

What it checks — and what it doesn’t

Format & checksum

PAN structure (10 chars, holder category) and the Aadhaar 12-digit Verhoeff checksum, validated deterministically in Core Java.

On-device OCR

Scan a PAN or Aadhaar image. Tesseract.js runs in your browser; the image is processed locally and never uploaded.

No database

No accounts table, no validation history, no document store. The session cookie carries only minimal signed claims.

Core Java core

A self-contained com.sun.net.httpserver server. No framework, no external state, deterministic output for the same input.

Format validation is not identity verification. This tool only checks structure and the Aadhaar Verhoeff checksum. It cannot confirm whether a number was actually issued by the Income Tax Department or UIDAI. Entered numbers and images are never stored, logged, or persisted.
